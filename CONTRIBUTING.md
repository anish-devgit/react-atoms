# Contributing to ReactAtoms

Thank you for considering contributing to ReactAtoms! This document provides guidelines for contributing.

## How to Contribute

### Reporting Bugs

- Use the GitHub issue tracker
- Describe the bug clearly with steps to reproduce
- Include browser/environment details
- Add screenshots if applicable

### Suggesting Features

- Open an issue with the `enhancement` label
- Describe the component/feature and its use case
- Explain why it would be valuable

### Submitting Pull Requests

1. **Fork the repository**
2. **Create a feature branch**: `git checkout -b feature/my-component`
3. **Commit changes**: `git commit -m "feat: add new component"`
4. **Push to branch**: `git push origin feature/my-component`
5. **Open a Pull Request**

### Component Guidelines

- Use TypeScript for type safety
- Include prop documentation
- Add demo implementation in `/src/components/demos`
- Ensure accessibility (ARIA labels, keyboard navigation)
- Test on multiple browsers
- Follow existing code style

### Commit Convention

Use conventional commits:
- `feat:` - New features
- `fix:` - Bug fixes
- `docs:` - Documentation updates
- `style:` - Code style changes
- `refactor:` - Code refactoring
- `test:` - Adding tests
- `chore:` - Maintenance tasks

## Code of Conduct

Please read [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) before contributing.

## Questions?

Open a discussion or reach out via issues.
